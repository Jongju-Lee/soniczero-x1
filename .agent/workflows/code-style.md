# 코드 스타일 분석 및 적용 규칙

**모든 코드 수정은 반드시 기존 스타일 분석 후** 그 스타일에 맞춰 작업해야 한다.

---

## 1단계: 스타일 분석 (수정 전 필수)

코드 수정 전, 해당 파일 및 관련 파일들을 먼저 읽고 아래 항목들을 파악한다.

### 공통 분석 항목
- [ ] **들여쓰기**: 스페이스 또는 탭? 몇 칸?
- [ ] **따옴표**: 작은따옴표(') vs 큰따옴표(")
- [ ] **세미콜론**: 사용 여부 (JavaScript)
- [ ] **후행 쉼표**: 마지막 요소에 쉼표 사용 여부
- [ ] **줄바꿈**: 선언문 사이 빈 줄 패턴

### JSX/TSX (HTML) 분석 항목
- [ ] **속성 순서**: className, id, data-*, aria-*, onClick 등의 순서
- [ ] **JSX 속성 따옴표**: JSX 속성(Attributes)에는 HTML과의 일관성을 위해 규칙적으로 **쌍따옴표(`"`)** 사용, JSX 중괄호 `{}` 내부나 일반 JS 로직 내의 문자열에는 주로 **홑따옴표(`'`)** 사용
- [ ] **들여쓰기**: 중첩 요소의 들여쓰기 패턴
- [ ] **주석 스타일**: {/* */} 형식 및 위치

### SCSS 분석 항목 (상세 규칙은 `/scss-formatting` 참조)
- [ ] **Nesting(중첩)**: 최대 중첩 깊이 (일반적으로 3 depth 제한), `&` 사용 패턴
- [ ] **선택자 간 빈 줄**: 같은 파트 내 셀렉터 간 빈 줄 패턴
- [ ] **파트 구분**: 상위 루트(/컴포넌트) 변경 시 빈 줄 및 주석
- [ ] **속성 순서**: Position → Box Model → Visual → Typography → Animation → Misc
- [ ] **변수 및 믹스인**: SCSS 변수(`$var`), CSS 변수(`var(--var)`), `@mixin`, `@include` 사용 패턴
- [ ] **모듈화**: SCSS 분할/불러오기 시 `@import` 절대 금지, **반드시 `@use`만 사용**

### React / JavaScript 분석 항목
- [ ] **컴포넌트 선언**: 화살표 함수 vs function 키워드
- [ ] **이벤트 핸들러**: 핸들러 네이밍 컨벤션 (예: handle*, on*)
- [ ] **상태 관리**: useState, useEffect 등 Hook 사용 패턴
- [ ] **변수 선언**: const/let 선호도
- [ ] **주석**: JSDoc 스타일 또는 인라인 주석
- [ ] **조건부 렌더링**: && 연산자 vs 삼항연산자 사용 패턴

---

## 2단계: 스타일 적용

분석된 스타일을 기반으로 코드를 작성한다.

### 반드시 따라야 할 규칙
1. **기존 패턴 우선**: 새로운 패턴 도입 금지, 기존 패턴 따르기
2. **일관성 유지**: 파일 내, 그리고 프로젝트 전체의 일관성 유지
3. **네이밍 컨벤션**: 기존 네이밍 규칙 분석 후 적용 (BEM, camelCase 등)
4. **주석 스타일**: 기존 주석 형식과 언어(한국어/영어) 따르기

### 판단이 어려운 경우
- 해당 파일 전체를 스캔하여 가장 많이 사용되는 패턴 채택
- 프로젝트 내 유사한 파일들 참조
- 판단이 여전히 어려우면 사용자에게 질문

## 3단계: 검증

수정 완료 후 다음을 확인한다.

- [ ] 기존 코드와 시각적으로 일관성 있게 보이는가?
- [ ] 새로운 패턴이 불필요하게 도입되지 않았는가?
- [ ] 들여쓰기, 빈 줄, 따옴표 등이 일관되게 적용되었는가?
